Gitflow Workflow

The Gitflow Workflow is an ideology and a set of operational best practices for DevOps teams. It is suited for projects with scheduled release cycles that require continuous releases.

Gitflow is the idea of a workflow for Git projects. The workflow specifies the kinds of branches that should be used and how they should be merged together. The gitflow is basically a set of command line tools that has an installation process. The installation process for git-flow is straightforward. Packages for git-flow are available on multiple operating systems. On OSX systems, you can execute brew install git-flow. On windows you will need to download and install git-flow. After installing git-flow you can use it in your project by executing git flow init. Git-flow is a wrapper around Git. The git flow init command is an extension of the default git init command and doesn't change anything in your repository other than creating branches for you.
