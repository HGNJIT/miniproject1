Gitflow Workflow

What is Gitflow

The Gitflow Workflow is an ideology and a set of operational best practices for DevOps teams. It is a branch-based flow specifically suited for teams and projects with regularly scheduled release cycle.

Gitflow is the idea of a workflow for Git projects. The workflow specifies the kinds of branches that should be used and how they should be merged together. The gitflow is basically a set of command line tools and their corresponding installation processes. The installation process for git-flow is extremely easy and straightforward. You can download the packages on almost all operating systems. After installing gitflow you can use it in your project by executing git flow init.

The basic idea of Gitflow is to allow for a divide and conquer approach to development. With the project broken out into multiple parts, different people or groups can work on individual features or ideas without interfering with the main project or each other. This makes it easier for teams to collaborate and allows for the easy distribution of work among large groups as well as small teams.

Gitflow steps:

  1. A sub-branch called develop is created from the master branch
  2. Within the develop branch, a release branch is created
  3. Inside of the release branch, multiple feature branches are created
  4. When a feature is ready for deployment, it is merged into the develop branch
  5. When the release branch is done it is merged into develop and master branches
  6. If an issue in master is detected a hotfix branch is created from master
  7. Once the hotfix is complete, it is merged to both develop and master
